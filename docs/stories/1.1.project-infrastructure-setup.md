## Dev Notes
This story establishes the foundational infrastructure for the Strategy Lab high-performance futures trading backtesting framework. The project targets single-user research environment with specific performance requirements.

**Key Architecture Context:**
- **Technology Stack**: Python 3.12+, uv package manager, hftbacktest core engine
- **Performance Targets**: 100K-500K ticks/second processing, 64GB RAM efficiency
- **Source Structure**: Follows docs/architecture/source-tree.md specification
- **Package Organization**: src/strategy_lab/ with modular components (core, data, strategies, backtesting, optimization, analysis, utils)

**Development Standards:**
- **Code Style**: Black formatter, 88 character line limit, double quotes preferred
- **Quality Tools**: Ruff for linting/imports, mypy for type checking
- **Testing**: pytest framework with comprehensive coverage requirements
- **Documentation**: Google-style docstrings, comprehensive API documentation

**Configuration Requirements:**
- **pyproject.toml**: Must include all dependencies from tech-stack.md
- **Development Dependencies**: Include black>=23.0.0, ruff>=0.1.0, mypy>=1.5.0, pytest>=7.0.0
- **Core Dependencies**: hftbacktest>=2.0.0, pandas>=2.0.0, pyarrow>=14.0.0, numpy>=1.24.0
- **Optimization**: scipy>=1.11.0, deap>=1.4.0

**Critical Project Structure:**
```
strategy_lab/
├── src/strategy_lab/           # Main package
├── tests/                      # Test suite
├── docs/                       # Documentation
├── data/                       # Data files (gitignored)
├── configs/                    # Configuration templates
├── scripts/                    # Utility scripts
└── results/                    # Backtest results (gitignored)
```
