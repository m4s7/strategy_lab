## Dev Notes
This story implements the first concrete strategy using Level 2 order book data, serving as both a working strategy and a validation of the framework's capabilities.

**Strategy Implementation Context:**
- **Module**: src/strategy_lab/strategies/implementations/order_book_imbalance.py
- **Base Class**: Inherits from StrategyBase
- **Data Requirements**: Level 2 order book depth data
- **Strategy Type**: Scalping strategy using market microstructure

**Order Book Imbalance Algorithm:**
- **Basic Formula**: (Bid Volume - Ask Volume) / (Bid Volume + Ask Volume)
- **Depth Weighting**: Weight deeper levels less than top-of-book
- **Smoothing**: Apply rolling average to reduce noise
- **Normalization**: Scale imbalance to [-1, 1] range

**Signal Generation Logic:**
- **Long Entry**: Imbalance > positive_threshold (e.g., 0.3)
- **Short Entry**: Imbalance < negative_threshold (e.g., -0.3)
- **Exit**: Imbalance returns to neutral zone or stop-loss triggered
- **Market Conditions**: Filter signals based on volatility and spread

**Position Sizing Methods:**
- **Fixed Size**: Constant position size per trade
- **Volatility Adjusted**: Size based on recent price volatility
- **Risk Budget**: Size based on maximum acceptable loss
- **Account Percentage**: Size as percentage of account equity

**Risk Management Features:**
- **Stop Loss**: Price-based or percentage-based stops
- **Time Stops**: Maximum holding period limits
- **Drawdown Protection**: Reduce size after losses
- **Market Hours**: Avoid signals near session boundaries

**Configuration Parameters:**
```yaml
strategy:
  name: "order_book_imbalance"
  parameters:
    positive_threshold: 0.3
    negative_threshold: -0.3
    smoothing_window: 5
    position_size: 1
    stop_loss_pct: 0.5
    max_holding_seconds: 300
```
