## Dev Notes
This story creates the foundational strategy template that all trading strategies will inherit from, providing a consistent interface and common functionality.

**Strategy Framework Context:**
- **Core Module**: src/strategy_lab/strategies/base/
- **Key Components**: strategy.py, signal_generator.py, position_manager.py
- **Design Pattern**: Template method pattern with abstract base class
- **Integration**: Works with hftbacktest engine and data pipeline

**Strategy Interface Design:**
- **Abstract Methods**: Requires implementation of core trading logic
- **Hook Methods**: Optional overrides for customization
- **Data Access**: Standardized methods for L1/L2 data consumption
- **Event Handling**: Tick processing, order events, market events

**Data Access Patterns:**
- **L1 Data**: `get_last_price()`, `get_best_bid()`, `get_best_ask()`, `get_volume()`
- **L2 Data**: `get_order_book()`, `get_depth_levels()`, `get_imbalance()`
- **Historical**: `get_price_history()`, `get_volume_profile()`
- **Market State**: `get_trading_session()`, `get_market_status()`

**Order Management Features:**
- **Order Types**: Market, limit, stop orders
- **Position Tracking**: Current position, P&L, exposure
- **Risk Controls**: Position limits, stop losses, maximum drawdown
- **Order Validation**: Price/size validation, market hour checks

**State Management Design:**
- **Strategy State**: Custom state variables and indicators
- **Persistence**: State maintained across tick processing
- **Serialization**: Save/restore state for backtesting
- **Validation**: Ensure state consistency and integrity

**Configuration System:**
- **Parameter Definition**: Type-safe parameter declarations
- **Validation**: Range checks, dependency validation
- **Hot Reload**: Runtime parameter updates during backtesting
- **Environment Support**: Different configs for dev/test/prod

**Template Features:**
- **Logging**: Integrated logging for strategy decisions
- **Metrics**: Built-in performance tracking
- **Debugging**: Debug hooks and state inspection
- **Testing**: Unit test helpers and mock data support
