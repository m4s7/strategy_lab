# Story 5.1: Data Pipeline Integration

## Status
Ready for Review

## Story
**As a** quantitative trader using Strategy Lab,
**I want** the backtesting engine to load and stream MNQ tick data from Parquet files,
**so that** strategies can execute against real historical market data instead of placeholder results.

## Acceptance Criteria

1. **Data Discovery**: System automatically discovers available MNQ contract data files in the configured data directory structure
2. **Parquet Loading**: System successfully loads MNQ tick data from Parquet files with proper schema validation
3. **Data Streaming**: Implements tick-by-tick data streaming interface for strategy consumption
4. **Date Range Filtering**: Filters data based on configured start_date and end_date from backtest configuration
5. **Contract Selection**: Processes only the contracts specified in the configuration (e.g., "03-24")
6. **Error Handling**: Provides clear error messages when data files are missing or corrupted
7. **Memory Management**: Efficiently handles large datasets without exceeding configured memory limits
8. **Integration**: Seamlessly integrates with existing BacktestEngine without breaking current CLI interface

### Expected Data Schema
MNQ tick data contains Level 1 and Level 2 market data with the following key fields:
- `timestamp`: Nanosecond precision timestamps
- `price`: Decimal128 with 2 decimal places
- `volume`: Int32 trade/quote volume
- `mdt`: Market Data Type (0=Ask, 1=Bid, 2=Last, etc.)
- `level`: Data level indicator (1 or 2)
- `operation`: Order book operations (0=Add, 1=Update, 2=Remove)
- `depth`: Order book depth level

### Integration Requirements
- Must work with existing configuration loader that provides data_path and contracts list
- Should leverage existing `data/ingestion/file_discovery.py` and `data_validator.py` modules
- Maintain compatibility with current CLI that expects BacktestEngine() initialization
