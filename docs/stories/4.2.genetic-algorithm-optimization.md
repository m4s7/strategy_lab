## Dev Notes
This story implements genetic algorithm optimization using DEAP library for efficient exploration of large parameter spaces where grid search is impractical.

**Genetic Algorithm Architecture:**
- **Core Module**: src/strategy_lab/optimization/algorithms/genetic_algorithm.py
- **Framework**: DEAP (Distributed Evolutionary Algorithms in Python)
- **Integration**: Works with strategy framework and backtesting engine
- **Scalability**: Supports parallel fitness evaluation

**DEAP Framework Integration:**
```python
from deap import base, creator, tools, algorithms

# Define fitness function for single-objective optimization
creator.create("FitnessMax", base.Fitness, weights=(1.0,))
creator.create("Individual", list, fitness=creator.FitnessMax)

# For multi-objective optimization (profit vs risk)
creator.create("FitnessMulti", base.Fitness, weights=(1.0, -1.0))  # Max profit, min risk
```

**Chromosome Encoding:**
- **Real-valued**: Continuous parameters encoded as float genes
- **Integer**: Discrete parameters encoded as integer genes
- **Mixed**: Support for mixed parameter types in single chromosome
- **Bounds**: Automatic constraint enforcement during variation

**Genetic Operators:**
- **Selection**: Tournament, roulette wheel, rank-based selection
- **Crossover**: Uniform, single-point, blend crossover for real values
- **Mutation**: Gaussian, polynomial, uniform mutation
- **Replacement**: Generational, steady-state replacement strategies

**Multi-objective Optimization (NSGA-II):**
- **Pareto Ranking**: Non-dominated sorting of solutions
- **Diversity**: Crowding distance for solution diversity
- **Objectives**: Profit maximization, risk minimization, drawdown minimization
- **Trade-offs**: Balanced exploration of objective space

**Convergence Criteria:**
- **Generation Limit**: Maximum number of generations
- **Fitness Stagnation**: Stop when best fitness doesn't improve
- **Diversity Loss**: Terminate when population diversity drops too low
- **Target Fitness**: Stop when target fitness level is reached

**Configuration Example:**
```yaml
genetic_algorithm:
  population_size: 100
  generations: 50
  crossover_prob: 0.8
  mutation_prob: 0.2
  selection_method: "tournament"
  tournament_size: 3
  objectives:
    - name: "sharpe_ratio"
      weight: 0.6
      maximize: true
    - name: "max_drawdown"
      weight: 0.4
      maximize: false
```

**Progress Monitoring:**
- **Fitness Evolution**: Track best/average/worst fitness per generation
- **Population Diversity**: Monitor genetic diversity metrics
- **Convergence Analysis**: Detect convergence patterns
- **Real-time Visualization**: Live plots of optimization progress

**Constraint Handling:**
- **Boundary Constraints**: Automatic parameter range enforcement
- **Linear Constraints**: Support for linear parameter relationships
- **Custom Constraints**: User-defined constraint functions
- **Penalty Methods**: Handle constraint violations with fitness penalties
