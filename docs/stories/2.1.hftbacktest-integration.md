## Dev Notes
This story establishes the core backtesting engine integration using hftbacktest for high-performance tick-by-tick simulation.

**hftbacktest Architecture Context:**
- **Core Module**: src/strategy_lab/backtesting/hft_integration/
- **Key Components**: adapter.py, data_feed.py, event_processor.py
- **Integration**: Bridge between Strategy Lab and hftbacktest library
- **Performance Target**: 100K+ ticks/second processing speed

**MNQ Contract Specifications:**
- **Symbol**: MNQ (Micro E-mini NASDAQ-100)
- **Tick Size**: $0.25 (varies by price level)
- **Contract Size**: $2 × Index Value
- **Trading Hours**: 6:00 PM - 5:00 PM ET (next day)
- **Commission**: Configurable per contract

**Data Format Requirements:**
- **Input**: MNQ tick data from pipeline (nanosecond timestamps)
- **Output**: hftbacktest compatible format
- **Conversion**: Price/volume scaling, timestamp normalization
- **Validation**: Data integrity checks during conversion

**Market Simulation Features:**
- **Order Types**: Market, limit, stop orders
- **Execution Logic**: Realistic price improvement and slippage
- **Latency Modeling**: Configurable order processing delays
- **Market Impact**: Price impact based on order size

**Performance Optimization:**
- **Data Feeding**: Efficient tick data streaming to hftbacktest
- **Memory Management**: Minimize allocations during processing
- **Parallel Processing**: Utilize multiple cores when possible
- **Caching**: Cache frequently accessed data structures

**Error Handling Patterns:**
- **Data Errors**: Invalid tick data, missing fields
- **Engine Errors**: hftbacktest internal errors
- **Resource Errors**: Memory/CPU exhaustion
- **Recovery**: Graceful degradation and checkpoint recovery
