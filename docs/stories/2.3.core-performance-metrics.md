# Story 2.3: Core Performance Metrics

## Status
Approved

## Story
**As a** quantitative trader,
**I want** comprehensive performance metrics calculated for each backtest,
**so that** I can evaluate strategy effectiveness and risk characteristics.

## Acceptance Criteria
1. System calculates total PnL, gross profit, and gross loss
2. Trade statistics include win rate, average win/loss, and trade count
3. Risk metrics include maximum drawdown, Sharpe ratio, and volatility
4. Time-based analysis shows monthly and daily performance breakdowns
5. Metrics are available both during backtesting and in final reports
6. Performance calculations handle edge cases like zero trades gracefully
7. Metrics are validated against known benchmarks for accuracy

## Dev Notes
This story implements the core performance measurement system that provides essential metrics for evaluating trading strategy effectiveness.

**Performance Metrics Architecture:**
- **Core Module**: src/strategy_lab/backtesting/metrics/
- **Key Components**: performance.py, risk.py, trade_analysis.py
- **Real-time Updates**: Metrics calculated incrementally during backtesting
- **Integration**: Works with backtest engine and reporting system

**Essential Metrics Categories:**
- **Profitability**: Total PnL, gross profit/loss, net profit margin
- **Trade Statistics**: Win rate, average trade, profit factor
- **Risk Measures**: Maximum drawdown, Sharpe ratio, volatility
- **Time Analysis**: Monthly/daily returns, performance stability

**P&L Calculation Details:**
- **Trade P&L**: Entry/exit price differences adjusted for position size
- **Commission Impact**: Realistic commission and fee deductions
- **Unrealized P&L**: Mark-to-market for open positions
- **Currency Handling**: Proper scaling for MNQ contract specifications

**Trade Statistics Formulas:**
- **Win Rate**: (Winning Trades / Total Trades) × 100
- **Average Win**: Total Winning Amount / Number of Winning Trades
- **Average Loss**: Total Losing Amount / Number of Losing Trades
- **Profit Factor**: Gross Profit / Gross Loss

**Risk Metrics Implementation:**
- **Maximum Drawdown**: Peak-to-trough decline in equity curve
- **Sharpe Ratio**: (Return - Risk-free Rate) / Standard Deviation
- **Volatility**: Standard deviation of returns over specified period
- **Value at Risk**: Statistical risk measure at confidence intervals

**Time-based Analysis:**
- **Monthly Returns**: Calendar month performance breakdown
- **Daily Analysis**: Day-of-week and time-of-day performance
- **Rolling Metrics**: Moving window calculations for trend analysis
- **Regime Analysis**: Performance across different market conditions

**Real-time Calculation Optimization:**
- **Incremental Updates**: Efficient metric updates per trade
- **Memory Efficiency**: Minimal memory footprint for long backtests
- **Performance**: Fast calculation suitable for real-time display
- **Accuracy**: Numerical precision handling for financial calculations

**Edge Case Handling:**
- **Zero Trades**: Appropriate default values and null handling
- **Division by Zero**: Safe mathematical operations
- **Data Quality**: Handle missing or invalid trade data
- **Outliers**: Robust calculation methods for extreme values
